# Submission Plan

Use this plan each time an assignment is implemented and prepared for upload.

## Standard Workflow

1. Open `docs/assignment_index.md` and choose the next assignment.
2. Read the assignment folder `README.md` and the matching PDF in `resources/source_pdfs/`.
3. Implement only inside the matching assignment folder.
4. Run the program or project locally and record validation notes in the assignment notebook file.
5. Add screenshots, screen recordings, or Drive links when the assignment requires them.
6. Create a submission ZIP from the assignment folder only.
7. Confirm the ZIP opens and contains all required files.
8. Update assignment status in docs if the implementation is complete.
9. Commit and push the repo so GitHub mirrors local progress.

## Evidence Rules

- For early script assignments, terminal output is enough unless the portal asks for screenshots.
- For PostgreSQL and OpenCV practical assignments, screenshots must show full code, output, and system timestamp.
- For the socket chat application, prepare both a ZIP and a complete screen recording.
- For Drive submissions, make the uploaded file or folder accessible before submitting the link.

## ZIP Naming

Use the portal-required format:

```text
YourName_AssignmentName.zip
```
